# PicRecog Development Standard (GEMINI.md)

이 문서는 PicRecog 프로젝트의 아키텍처 및 코딩 규약을 정의합니다. 모든 개발 및 AI 에이전트 작업의 최우선 지침입니다.

## 1. 아키텍처 개요
- **목적**: AI 기반 이미지 자동 분류 및 시맨틱(벡터) 검색 시스템.
- **스택**: Python, Streamlit, PostgreSQL (pgvector), SQLAlchemy, LangGraph.

## 2. 백엔드 핵심 규약

### [DI & DB] 자동 의존성 주입
- **`@transactional`**: `db: Session` 파라미터가 있는 서비스 함수에 필수 적용.
- **작동 원리**: `inspect.signature`를 통해 런타임에 세션을 주입하고 생명주기를 자동 관리.

### [AI & Client] SRP 기반 클라이언트 분리
- **StudioLLMClient**: VLM 기반 채팅 및 이미지 분류(JSON 출력) 전담.
- **StudioEmbeddingClient**: 순수 텍스트 임베딩 API 호출 전담.
- **EmbeddingClient**: 이미지 묘사(LLM)와 텍스트 벡터화(Embedding)를 결합하는 오케스트레이터.
- **AgentExecutorClient**: LangGraph 기반 도구 호출 및 지능형 검색 에이전트.

### [Agent & Tool] 지능형 검색 및 구조화된 출력
- **도구 호출 우선**: 시스템 프롬프트를 통해 '도구 선 호출 후 결과 보고' 순서 엄격 통제.
- **호환성 대응**: 로컬 LLM의 호환성을 위해 `create_agent`의 `response_format` 대신 시스템 프롬프트 지시와 `parse_structured_response` 유틸리티를 사용한 수동 파싱 적용.
- **엄격한 필터링**: 에이전트는 검색 도구의 결과를 반환하기 전, 사용자의 수량(예: 2명), 속성(색상, 행위) 제약 조건을 '설명' 필드와 대조하여 엄격히 필터링해야 함.
- **개별 판단 근거**: 검색 결과의 각 이미지마다 해당 이미지가 선택된 구체적이고 차별화된 이유(`justification`)를 반드시 포함하여 투명성 확보.

### [Async & Performance] 비동기 최적화
- **비차단 I/O**: 네트워크(API)는 `async/await`, CPU/파일 I/O는 `asyncio.to_thread()` 사용.
- **싱글톤 워크플로우**: 서비스 인스턴스 내에서 워크플로우를 1회 컴파일 후 재사용.
- **Batch 처리**: 엔진 레벨 병렬 처리를 위해 `abatch()` 메서드 적극 활용.

### [Utility] 공통 로직 자산화 (DRY)
- **`backend/system/utils.py`**: 도메인 중립적 범용 로직(응답 파싱 등)은 반드시 유틸리티로 분리.
- **참조 의무**: 신규 구현 전 기존 유틸리티 존재 여부를 먼저 확인하여 중복 코드 방지.

### [Test & Debug] 테스트 및 디버깅 규약
- **Mock 코드 금지**: 프로덕션 코드(서비스, UI 등) 내에 `mock:` 접두사 등을 이용한 더미 데이터 테스트 로직을 직접 삽입하지 않음.
- **테스트 분리**: 모든 테스트 및 검증 로직은 `tests/` 디렉토리 내의 독립된 테스트 모듈이나 전용 테스트 환경에서 수행.

## 3. UI (Streamlit) 핵심 규약

### [UX] 일관된 레이아웃 및 상태 관리
- **인라인 렌더링**: 상세 정보 및 유사 검색 결과는 해당 항목 하단에 행 단위로 표시.
- **최신 API**: `use_container_width` 대신 `width="stretch"` 사용.
- **동적 바인딩**: 모든 카테고리/이모지는 `shared/constants.py`를 참조하여 동적 렌더링.
- **분석 근거 표시**: 자연어 검색 결과 시 에이전트가 제공한 개별 `justification`을 사용자에게 명확히 노출.

### [Real-time] 비동기 UI 피드백
- **비차단 아키텍처**: `asyncio.Queue`와 별도 스레드를 사용하여 UI 리런 없이 작업 수행.
- **부분 갱신**: `st.fragment`를 활용하여 실시간 진행 상태(로그, 노드명)를 동적 표시.
- **멀티 태스크**: `UIEventHandler`를 통해 여러 백그라운드 작업을 동시에 추적 및 관리.
